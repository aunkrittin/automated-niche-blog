# เจาะลึก Windows QoS และ DSCP Packet Tagging: จัดลำดับความสำคัญทราฟฟิกเครือข่ายระดับนโยบายเพื่อสตรีมและเล่นเกมโดยไร้อาการปิงแกว่ง

สำหรับเกมเมอร์สาย Competitive และสตรีมเมอร์แล้ว ศัตรูตัวฉกาจไม่ใช่ฝีมือของคู่ต่อสู้ แต่คือ **Latency Spikes (อาการปิงแกว่ง)** และ **Packet Loss** ที่มักจะเกิดขึ้นอย่างกะทันหันในจังหวะสำคัญ หลายคนพยายามแก้ปัญหาด้วยการเปลี่ยนไปใช้แพ็กเกจอินเทอร์เน็ตที่เร็วขึ้น แต่ก็ยังพบปัญหาเดิม นั่นเป็นเพราะปัญหาไม่ได้อยู่ที่ "Bandwidth" (ความกว้างของท่อ) แต่อยู่ที่ "Traffic Congestion" และ "Bufferbloat" (การจัดคิวของข้อมูลที่ค้างสะสมในเลเยอร์เครือข่าย)

ในบทความเชิงลึกนี้ เราจะมาเจาะลึกกลไก **Windows Policy-based QoS (Quality of Service)** และการทำ **DSCP (Differentiated Services Code Point) Packet Tagging** ซึ่งเป็นการระบุเครื่องหมายบน IP Header ในระดับ Kernel ของ Windows เพื่อบอกให้เราเตอร์และอุปกรณ์เครือข่ายรู้ว่า Packet ไหนคือข้อมูลเกมที่ต้องส่งทันที และ Packet ไหนคือการดาวน์โหลดทั่วไปที่รอได้

---

## ทำความเข้าใจก่อนเริ่ม: DSCP และ QoS ทำงานอย่างไรในระดับ Layer 3?

ตามมาตรฐาน OSI Model ทราฟฟิกเครือข่ายทั้งหมดจะเดินทางผ่านโปรโตคอล TCP/IP ใน Layer 3 (Network Layer) ซึ่งใน IPv4 Header จะมีฟิลด์หนึ่งที่เรียกว่า **ToS (Type of Service)** 

ต่อมามาตรฐาน IETF ได้ปรับปรุงฟิลด์ ToS นี้ให้เป็น **DSCP (Differentiated Services Code Point)** ขนาด 6 บิต ซึ่งทำให้เราสามารถกำหนดระดับความสำคัญของแพ็กเก็ตข้อมูลได้ถึง 64 ระดับ (0 ถึง 63)

```
+-------------------------------------------------------------+
|                        IPv4 Header                          |
+-------------------+--------------------+--------------------+
|  Version / IHL    |  DSCP (6 bits) | ECN |    Total Length    |
+-------------------+--------------------+--------------------+
```

### ค่า DSCP ที่สำคัญสำหรับการสตรีมและการเล่นเกม:
1. **Expedited Forwarding (EF) - ค่า DSCP: 46 (Decimal)**: เป็นระดับความสำคัญสูงสุดสำหรับทราฟฟิกที่ต้องการความหน่วงต่ำสุด (Low Latency) และอัตราการสูญเสียข้อมูลต่ำสุด เหมาะอย่างยิ่งสำหรับ **ข้อมูลการเล่นเกม (Game Payload)** และ **VoIP (Discord/TeamSpeak)**
2. **Assured Forwarding (AF41) - ค่า DSCP: 34 (Decimal)**: เหมาะสำหรับทราฟฟิกประเภทวิดีโอสตรีมมิ่งที่ต้องการแบนด์วิดท์สูงแต่สม่ำเสมอ เช่น **OBS Studio (RTMP/SRT Streaming)**

เมื่อ Windows ทำการ "Tag" หรือติดป้ายกำกับค่าเหล่านี้ลงไปในทุกๆ แพ็กเก็ตที่ส่งออกจากเครื่องคอมพิวเตอร์ของเราเตอร์ที่รองรับ QoS จะอ่านค่านี้และจัดลำดับคิวการส่งออกไปยังอินเทอร์เน็ตเป็นอันดับแรกทันที (First-in, First-out ในคิวลำดับความสำคัญสูง) ทำให้ปิงไม่แกว่งแม้ว่าจะมีคนอื่นในบ้านกำลังดาวน์โหลดไฟล์ขนาดใหญ่อยู่ก็ตาม

> ⚠️ **ข้อควรระวังสำคัญ (CRITICAL WARNING):**
> การปรับแต่งระบบเครือข่ายในระดับนโยบาย (Group Policy) และการแก้ไข Registry มีความเกี่ยวข้องกับระบบภายในของ Windows OS โดยตรง แม้ว่าวิธีการด้านล่างนี้จะมีความปลอดภัยสูงและได้รับการพิสูจน์แล้วตามมาตรฐานสากล แต่เพื่อความปลอดภัยสูงสุดของระบบคอมพิวเตอร์ของคุณ **แนะนำให้สร้าง System Restore Point ก่อนเริ่มดำเนินการทุกครั้ง** 
>
> **วิธีสร้าง Restore Point:** กดปุ่ม `Win + S` พิมพ์คำว่า "Create a restore point" เลือกไดรฟ์ระบบ (ปกติคือ C:) คลิก "Create" และตั้งชื่อจุดบันทึกข้อมูลก่อนกดยืนยัน

## ขั้นตอนที่ 1: การเปิดใช้งาน DSCP Tagging บน Windows (ผ่าน Group Policy)

สำหรับผู้ใช้งาน Windows Pro, Enterprise หรือ Education เราสามารถกำหนดนโยบาย QoS ได้โดยตรงผ่าน Local Group Policy Editor

1. กดปุ่ม `Windows + R` พิมพ์ `gpedit.msc` แล้วกด Enter
2. นำทางไปยังโฟลเดอร์ต่อไปนี้ในแถบซ้ายมือ:
   `Computer Configuration` -> `Windows Settings` -> คลิกขวาที่ `Policy-based QoS`
3. เลือก **Create new policy...**

### การตั้งค่านโยบายสำหรับเกมเมอร์ (เช่น Valorant, Warzone, CS2)
เพื่อลดอาการปิงแกว่งในเกม เราจะส่งข้อมูลเกมด้วยลำดับความสำคัญสูงสุด (EF - 46)

* **Policy name:** ตั้งชื่อนโยบาย (เช่น `Gaming_Low_Latency`)
* **DSCP Value:** ติ๊กเลือก Specify DSCP Value และใส่ค่า `46`
* คลิก **Next**
* เลือก **Only applications with this executable name** แล้วใส่ชื่อไฟล์ `.exe` ของเกมที่คุณต้องการ (เช่น `Valorant.exe` หรือ `cod.exe`)
* คลิก **Next**
* เลือก **Any source IP address** และ **Any destination IP address** แล้วคลิก **Next**
* เลือกโปรโตคอลเป็น **UDP** (เนื่องจากเกมออนไลน์ส่วนใหญ่ส่งข้อมูลพิกัดและสถานะผ่าน UDP เพื่อความรวดเร็ว) และเลือก **From any source port** และ **To any destination port**
* คลิก **Finish**

### การตั้งค่านโยบายสำหรับสตรีมเมอร์ (OBS Studio)
เพื่อป้องกันไม่ให้การสตรีมสะดุดหรือเฟรมเรตร่วง (Dropped Frames) เมื่อเครือข่ายในบ้านมีความหนาแน่น

* **Policy name:** ตั้งชื่อนโยบาย (เช่น `OBS_Streaming`)
* **DSCP Value:** ติ๊กเลือก Specify DSCP Value และใส่ค่า `34` (AF41)
* คลิก **Next**
* เลือก **Only applications with this executable name** ใส่ค่า `obs64.exe`
* คลิก **Next** ไปจนถึงหน้าสุดท้าย
* เลือกโปรโตคอลเป็น **TCP** (เนื่องจากการสตรีมทั่วไปใช้ RTMP ซึ่งทำงานบน TCP) หรือเลือก **TCP and UDP** หากใช้โปรโตคอลใหม่อย่าง SRT
* คลิก **Finish**

## ขั้นตอนที่ 2: การแก้ไข Registry สำหรับ Windows Home Edition และการเปิดใช้แบบ Non-Domain

สำหรับ Windows Home Edition จะไม่มีเครื่องมือ `gpedit.msc` นอกจากนี้ใน Windows รุ่นทั่วไปที่ไม่ได้เชื่อมต่อกับ Domain (Active Directory) ระบบจะไม่ยอมส่งค่า DSCP ออกไปกับแพ็กเก็ตภายนอกเครื่องตามปกติ เราจึงจำเป็นต้องเปิดใช้งานระบบนี้ผ่าน Windows Registry

```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\QoS]
"DoNotUseNLA"="1"
```

### ขั้นตอนการดำเนินการด้วยตนเองอย่างปลอดภัย:
1. กดปุ่ม `Windows + R` พิมพ์ `regedit` แล้วกด Enter
2. นำทางไปยังตำแหน่งคีย์ดังนี้:
   `HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip`
3. ตรวจสอบว่ามีโฟลเดอร์ (Key) ที่ชื่อว่า `QoS` หรือยัง หากไม่มีให้คลิกขวาที่โฟลเดอร์ `Tcpip` เลือก **New** -> **Key** แล้วตั้งชื่อว่า `QoS`
4. คลิกเข้าไปที่โฟลเดอร์ `QoS` คลิกขวาที่พื้นที่ว่างทางขวา เลือก **New** -> **String Value** (ค่าสตริง)
5. ตั้งชื่อค่านี้ว่า `DoNotUseNLA`
6. ดับเบิ้ลคลิกที่ `DoNotUseNLA` และใส่ค่าในช่อง Value data เป็น `1` จากนั้นกด OK
7. ทำการ **Restart คอมพิวเตอร์** เพื่อให้ระบบโหลดค่า TCP/IP Stack ใหม่

## ขั้นตอนที่ 3: การตรวจสอบด้วย Wireshark (การวิเคราะห์ Packet เชิงลึก)

เพื่อพิสูจน์ว่า Windows ทำการ Tag Packet ให้เราจริงหรือไม่ เราสามารถตรวจสอบการทำงานได้ด้วยเครื่องมือวิเคราะห์ทราฟฟิกระดับโลกอย่าง **Wireshark**

1. ดาวน์โหลดและติดตั้ง Wireshark จากนั้นเปิดโปรแกรมและเลือกการ์ดเครือข่ายที่ใช้งาน (เช่น Ethernet) เพื่อเริ่มดักจับข้อมูล (Capture)
2. เปิดเกมของคุณหรือเปิดโปรแกรม OBS Studio แล้วกดเริ่มสตรีมเพื่อให้เกิดทราฟฟิกข้อมูล
3. ในช่อง Filter ของ Wireshark ด้านบน ให้พิมพ์คำสั่งกรองข้อมูล:
   ```text
   ip.dsfield.dscp == 46
   ```
   หรือสำหรับ OBS:
   ```text
   ip.dsfield.dscp == 34
   ```
4. หากกำหนดค่าถูกต้อง คุณจะเห็นรายการ Packet ปรากฏขึ้นมา เมื่อคลิกดูรายละเอียดที่ส่วนของ **Internet Protocol Version 4** คุณจะเห็นฟิลด์:
   `Differentiated Services Field: 0xb8 (DSCP: EF, ECN: Not-ECT)` แสดงว่าระบบเริ่ม Tag ข้อมูลเรียบร้อยแล้ว!

```
Frame 1: 98 bytes on wire
Internet Protocol Version 4, Src: 192.168.1.50, Dst: 24.105.29.30
    0100 10.. = Differentiated Services Codepoint: Expedited Forwarding (46)
    .... ..00 = Explicit Congestion Notification: Not-ECT (0)
```

## ข้อจำกัดที่ควรรู้และฮาร์ดแวร์ที่แนะนำเพื่อรีดประสิทธิภาพสูงสุด

แม้ว่าคุณจะตั้งค่าใน Windows อย่างสมบูรณ์แบบแล้ว แต่หากข้อมูลเหล่านี้ส่งไปถึง **เราเตอร์ (Router)** ที่ไม่มีประสิทธิภาพ หรือสายสัญญาณอินเทอร์เน็ตภายในบ้านมีสัญญาณรบกวน ค่า DSCP Tagging ก็จะสูญเปล่า เนื่องจากเราเตอร์ระดับล่างมักจะละเลย (Ignore) หรือลบค่า DSCP ทิ้ง (Remark to 0)

เพื่อแก้ปัญหานี้อย่างยั่งยืน อุปกรณ์ฮาร์ดแวร์ในบ้านของคุณต้องรองรับการอ่านค่า DSCP/802.1p และมีการรับส่งข้อมูลที่นิ่งและเสถียรที่สุด

### 1. สาย LAN คุณภาพสูงที่มีการซีลป้องกันสัญญาณรบกวน (Shielded Twisted Pair)
การเล่นเกมโดยใช้ Wi-Fi มักจะเกิดสภาวะ Packet Loss จากสัญญาณรบกวนในอากาศ การเปลี่ยนมาใช้สาย LAN เกรดอุตสาหกรรมที่มีมาตรฐาน Cat6 หรือ Cat8 เป็นการลงทุนที่คุ้มค่าที่สุดในการตัดปัญหาเรื่องปิงแกว่ง
* [ดูรายละเอียดอุปกรณ์หรือสั่งซื้อสาย LAN เกรดพรีเมียมได้ที่นี่](https://shopee.co.th/search?keyword=Cat8+LAN+Cable+Ugreen)

### 2. เราเตอร์เกรดเกมมิ่งที่รองรับระบบ Advanced QoS และ Bandwidth Control
เราเตอร์ระดับทั่วไปจะไม่สามารถจัดคิวการรับส่งข้อมูลได้อย่างมีประสิทธิภาพเมื่อมีอุปกรณ์เชื่อมต่อพร้อมกันหลายเครื่อง เกมมิ่งเราเตอร์ระดับไฮเอนด์จะมีฟังก์ชันในการสแกนหา DSCP Packets และจัดลำดับความสำคัญให้เหนือกว่าอุปกรณ์สมาร์ทโฟนหรือสมาร์ททีวีในบ้านโดยอัตโนมัติ
* [ดูรายละเอียดเราเตอร์ระดับ Gaming ประสิทธิภาพสูงได้ที่นี่](https://shopee.co.th/search?keyword=ASUS+ROG+Gaming+Router)

## บทสรุป

การปรับแต่ง **Windows Policy-based QoS** ร่วมกับการตั้งค่า **DSCP Packet Tagging** เป็นหนึ่งในเทคนิคการปรับแต่งระบบเครือข่ายขั้นสูงที่มีประสิทธิภาพสูงสุด เพราะเป็นการแก้ปัญหาอาการปิงแกว่งที่ต้นตอของระบบปฏิบัติการ โดยไม่ต้องพึ่งพาซอฟต์แวร์ลดปิงราคาแพง เมื่อผนวกเข้ากับระบบ Network Hardware ที่ดีภายในบ้าน คุณก็พร้อมที่จะลุยทุกสมรภูมิออนไลน์และการสตรีมมิ่งคุณภาพสูงได้อย่างลื่นไหลไร้รอยต่อแน่นอน!
