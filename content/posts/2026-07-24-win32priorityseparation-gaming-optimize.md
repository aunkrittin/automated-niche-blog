# ถอดรหัส Win32PrioritySeparation: ปรับแต่งระบบจัดสรรเวลาประมวลผล CPU เพื่อลด Micro-Stuttering และเพิ่มความลื่นไหลในเกมขั้นสุด

เคยไหม? เฟรมเรต (FPS) ในเกมแสดงผลสูงทะลุ 140+ FPS แต่ความรู้สึกขณะขยับเมาส์หรือหันมุมกล้องกลับไม่ลื่นไหลอย่างที่ควรจะเป็น มีอาการกระตุกกึกๆ เล็กๆ เป็นระยะที่เรียกว่า **Micro-Stuttering** หรือ **Frametime Spikes** 

ในฐานะ System Architect และ Game Server Developer ปัญหาเหล่านี้มักไม่ได้เกิดจากความแรงของฮาร์ดแวร์ไม่เพียงพอ แต่เกิดจาก **"Windows Thread Scheduler"** หรือระบบจัดสรรคิวงานของ CPU ที่พยายามแบ่งปันเวลาประมวลผลให้โปรแกรมเบื้องหลัง (Background Services) มากเกินไปในขณะที่คุณกำลังเล่นเกม

บทความนี้จะพาคุณไปเจาะลึกกลไกภายในของระบบปฏิบัติการ Windows และเปิดเผยวิธีการปรับแต่งค่าฟังก์ชันลับอย่าง **Win32PrioritySeparation** เพื่อบังคับให้ CPU มุ่งเน้นทรัพยากรไปที่ตัวเกมแบบ 100% เพื่อลดความหน่วงและสร้าง Frame Pacing ที่นิ่งสนิทราวกับระนาบเส้นตรง

---

## 1. เข้าใจกลไก Windows Thread Scheduler และปัญหา Micro-Stuttering

สถาปัตยกรรมของ Windows (Client OS เช่น Windows 10 และ 11) ถูกออกแบบมาให้ทำงานในลักษณะ **Multitasking** เป็นหลัก ซึ่งหมายความว่าระบบปฏิบัติการต้องเฉลี่ยเวลาประมวลผลของ CPU ให้กับทุกๆ โปรแกรมที่เปิดอยู่ รวมถึงระบบ System Services ของ Windows เองด้วย

หน่วยเวลาที่ CPU มอบให้กับแต่ละ Thread ในการประมวลผลเรียกว่า **"Quantum"** (ไม่ใช่ฟิสิกส์ควอนตัม แต่เป็นช่วงเวลาสั้นๆ มักวัดเป็นหน่วย Clock Ticks ของระบบ)

```
[ Thread Scheduler ] ───► แบ่งเวลาประมวลผล (Quantum) ───► [ Game Thread ] (ต้องการความนิ่ง)
                                                    ───► [ Background App ] (เช่น Discord, Chrome)
```

เมื่อคุณเล่นเกมที่ต้องการการประมวลผลแบบ Real-time หาก Windows Scheduler ตัดสินใจ "ขัดจังหวะ" (Context Switch) เพื่อเอาเวลาของ CPU ไปให้โปรแกรมเบื้องหลังประมวลผล แม้จะเพียงแค่เสี้ยววินาที ผลลัพธ์ที่ตามมาคือ **Frametime Spike** (เวลาในการเรนเดอร์เฟรมนั้นๆ สูงขึ้นเฉียบพลัน) ซึ่งส่งผลให้เกิด Micro-Stuttering ในเกมทันที

## 2. เจาะลึก Win32PrioritySeparation คืออะไร?

`Win32PrioritySeparation` คือค่า Registry ในระบบปฏิบัติการ Windows (อยู่ที่พาธ `HKLM\SYSTEM\CurrentControlSet\Control\PriorityControl`) ทำหน้าที่ควบคุมกลยุทธ์การจัดสรร CPU Quantum โดยทำงานผ่านบิตมาสก์ (Bitmask) ขนาด 6 บิต (บิต 0 ถึง 5) แบ่งออกเป็น 3 ส่วนสำคัญ:

```
┌─────────────────────────── Win32PrioritySeparation (6-Bit Mask) ───────────────────────────┐
│  Bits 5-4: Quantum Length     │   Bits 3-2: Quantum Type       │  Bits 1-0: Foreground Boost  │
│  01: Short (เกมเมอร์ต้องการ)   │   01: Variable (ค่าเริ่มต้น)    │  00: No Boost                │
│  10: Long                     │   10: Fixed (เกมเมอร์ต้องการ)   │  01: Medium Boost (2:1)      │
│                               │                                │  10: High Boost (3:1)        │
└────────────────────────────────────────────────────────────────────────────────────────────┘
```

### การจัดอันดับคุณลักษณะที่ส่งผลต่อการเล่นเกม:

1. **Quantum Length (ความยาวของรอบการประมวลผล):**
   * **Short (สั้น):** เหมาะสำหรับการตอบสนองที่รวดเร็ว (Interactive/Gaming) ระบบจะสลับงานอย่างรวดเร็ว ทำให้ค่า Latency ต่ำลง
   * **Long (ยาว):** เหมาะสำหรับงานคำนวณหนักๆ เบื้องหลัง (Server/Rendering) ป้องกันการสูญเสียเวลาจากการ Context Switching บ่อยๆ

2. **Quantum Type (ความยืดหยุ่นของเวลา):**
   * **Variable (แปรผัน):** Windows จะปรับเปลี่ยนขนาดเวลาประมวลผลตามสถานะของหน้าต่างที่โฟกัส (มักทำให้ Frame Pacing ไม่นิ่ง)
   * **Fixed (คงที่):** กำหนดเวลาประมวลผลให้คงที่อย่างเข้มงวด ส่งผลให้การจ่ายงานให้ CPU มีความสม่ำเสมอสูงมาก ลดอาการกระตุกเฉียบพลันได้อย่างมีนัยสำคัญ

3. **Foreground Boost (การจัดลำดับความสำคัญของแอปพลิเคชันหน้าแรก):**
   * กำหนดสัดส่วนพลังงานประมวลผลระหว่างแอปพลิเคชันที่คุณกำลังเล่น (Foreground) กับบริการเบื้องหลัง (Background)

## 3. ถอดรหัสค่าตัวเลขยอดนิยม: เลือกค่าไหนดีที่สุดสำหรับเครื่องคุณ?

ค่ามาตรฐานของ Windows Client ทั่วไปจะตั้งไว้ที่ **Hex `0x26` (Decimal 38)** ซึ่งหมายถึง *Short, Variable, High Boost (3:1)* แต่ค่าเหล่านี้ไม่ใช่ค่าที่ดีที่สุดสำหรับการเล่นเกมที่ต้องการความนิ่งของเฟรมเรตสูง

ด้านล่างนี้คือตารางค่าปรับแต่งที่ถูกพิสูจน์แล้วในวงการ Competitive Gaming:

| ค่าปรับแต่ง (Hex) | ค่าปรับแต่ง (Decimal) | ลักษณะการทำงาน (Quantum Configuration) | ผลลัพธ์ในทางปฏิบัติ |
| :--- | :--- | :--- | :--- |
| `0x26` *(Default)* | `38` | Short, Variable, High Boost (3:1) | ลื่นไหลทั่วไปสำหรับคอมใช้งานทั่วไป แต่ยังมีอาการสลัดเฟรม (Micro-Stutter) เมื่อมีแอปเบื้องหลังทำงาน |
| `0x28` *(แนะนำสูงสุด)* | `40` | **Short, Fixed, High Boost (3:1)** | **ดีที่สุดสำหรับการเล่นเกมส่วนใหญ่** เฟรมเรตนิ่งขึ้นอย่างเห็นได้ชัด เพราะใช้ Fixed Quantum ป้องกันการสลับเวลาแบบแปรปรวน |
| `0x16` | `22` | Short, Variable, Medium Boost (2:1) | เหมาะสำหรับสเปก CPU ระดับเริ่มต้น (Low-end Multi-core) ที่ต้องการความสมดุลระหว่างเกมและบราวเซอร์ |
| `0x18` | `24` | Long, Fixed, No Boost (1:1) | เป็นค่าเริ่มต้นของ Windows Server ไม่แนะนำสำหรับการเล่นเกมอย่างยิ่ง เพราะจะทำให้ Latency ของเมาส์และเกมสูงขึ้น |

⚠️ **กล่องข้อความแจ้งเตือนภัยและข้อควรระวัง (CRITICAL SAFETY WARNING)**

> **โปรดอ่านก่อนลงมือทำ:** การแก้ไข Registry ของระบบมีความเสี่ยงหากทำผิดพลาด อาจส่งผลทำให้ระบบปฏิบัติการทำงานไม่เสถียร หรือเกิดสภาวะระบบล่ม (BSOD) ได้ 
> 
> **สิ่งที่ต้องทำก่อนเริ่มขั้นตอนปรับแต่ง:**
> 1. ทำการสร้าง **System Restore Point** เสมอ โดยการพิมพ์คำว่า "Create a restore point" ในช่องค้นหาของ Windows แล้วคลิก Create
> 2. ทำการสำรองข้อมูล (Backup) ค่า Registry เดิมไว้ก่อนทำการแก้ไขทุกครั้ง

## 4. ขั้นตอนการปรับแต่งทีละขั้นตอน (Step-by-Step Guide)

เราสามารถเลือกวิธีปรับแต่งได้ 2 รูปแบบตามความสะดวกของคุณ:

### วิธีที่ 1: ปรับแต่งผ่าน Registry Editor (แนะนำสำหรับผู้เริ่มต้น)

1. กดปุ่ม `Windows + R` บนคีย์บอร์ด พิมพ์คำว่า `regedit` แล้วกด **Enter** เพื่อเปิด Registry Editor
2. นำทางไปยังตำแหน่งพาธต่อไปนี้ (คัดลอกไปวางในช่อง Address Bar ด้านบนได้):
   ```text
   HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl
   ```
3. ที่พาเนลด้านขวา ค้นหาคีย์ที่มีชื่อว่า `Win32PrioritySeparation`
4. ดับเบิ้ลคลิกที่คีย์ดังกล่าว จะมีหน้าต่างป๊อปอัปแสดงขึ้นมา
5. ในส่วนของ **Base** ให้เลือกเป็น **Hexadecimal** (เลขฐานสิบหก) หรือ **Decimal** (เลขฐานสิบ)
6. ทำการแก้ไขค่าเป็น:
   * หากเลือก Base เป็น **Hexadecimal** ให้ใส่ค่า: `28`
   * หากเลือก Base เป็น **Decimal** ให้ใส่ค่า: `40`
7. คลิก **OK** จากนั้นทำการปิดโปรแกรม Registry Editor และ **Restart คอมพิวเตอร์ของคุณ** เพื่อให้ระบบเปิดใช้การตั้งค่าใหม่

---

### วิธีที่ 2: ปรับแต่งด้วยสคริปต์อัตโนมัติ (สำหรับ Power Users)

เพื่อความสะดวกรวดเร็วและลดความผิดพลาดจากการพิมพ์ค่าผิด คุณสามารถใช้ Windows PowerShell หรือนำเข้าไฟล์ `.reg` ได้โดยตรง

#### โค้ดสำหรับไฟล์ Registry (.reg)
สร้างไฟล์ข้อความใหม่ผ่าน Notepad คัดลอกโค้ดด้านล่างนี้ไปวาง แล้วบันทึกไฟล์เป็นชื่อ `Optimize_Quantum.reg` (ตรวจสอบให้แน่ใจว่านามสกุลไฟล์ไม่ใช่ .txt) จากนั้นดับเบิ้ลคลิกเพื่อ Run:

```registry
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl]
"Win32PrioritySeparation"=dword:00000028
```

#### คำสั่งตรวจสอบผ่าน PowerShell (เช็คความถูกต้อง)
หลังจากรีสตาร์ทเครื่อง คุณสามารถตรวจสอบว่าค่าใหม่ถูกบันทึกเข้าระบบเรียบร้อยแล้วหรือไม่ โดยเปิด PowerShell แบบ Run as Administrator แล้วพิมพ์คำสั่งด้านล่างนี้:

```powershell
Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\PriorityControl" -Name "Win32PrioritySeparation"
```

*หากผลลัพธ์แสดงเป็น `Win32PrioritySeparation : 40` แสดงว่าการตั้งค่าสำเร็จลุล่วงด้วยดี!*

## 5. วิธีการทดสอบผลลัพธ์ (How to Benchmark and Verify)

การทดสอบประสิทธิภาพที่ได้มาตรฐานวิศวกรรม ไม่ใช่การนั่งมองแต่ค่าเฉลี่ย FPS (Average FPS) เนื่องจากตัวเลขเฉลี่ยจะไม่แสดงผลกระทบของ Micro-Stuttering แต่เราจะพิจารณาค่า **1% Low FPS** และกราฟ **Frame Pacing (Frame Delivery Time)**

### เครื่องมือแนะนำในการทดสอบ:
* **CapFrameX** หรือ **PresentMon (Intel):** เครื่องมือจับภาพสถิติแบบละเอียดระดับไมโครวินาที (Microseconds)
* **MSI Afterburner (RTX RivaTuner):** สำหรับดูหน้าจอตรวจวัดแบบ Real-time (มองหาเส้นกราฟ Frametime หากเส้นกราฟนิ่งเรียบไม่มีรอยหยักแหลมสูงขึ้น แสดงว่าแก้ปัญหาสำเร็จแล้ว)

```
[ Frametime Graph ] 
ก่อนปรับแต่ง (Default 0x26):  ──/\───/\──/\─── (มีรอยหยักสะดุดบ่อยครั้ง)
หลังปรับแต่ง (Optimized 0x28): ────────────── (เส้นกราฟตรง สม่ำเสมอ ลื่นไหลสูงสุด)
```

## 6. ฮาร์ดแวร์ซีเนอร์ยี่ (Hardware Synergy) เพื่อรีดประสิทธิภาพสูงสุด

การปรับแต่ง Thread Scheduler ในระดับซอฟต์แวร์ระบบปฏิบัติการจะมีประสิทธิภาพสูงสุดต่อเมื่อไม่มี "คอขวด (Bottlenecks)" ในส่วนของฮาร์ดแวร์ โดยมี 3 อุปกรณ์สำคัญที่ทำงานสอดประสานโดยตรงกับหน่วยเวลาประมวลผลของ CPU:

1. **หน่วยความจำ RAM ความหน่วงต่ำ (Low Latency RAM):**
   การจัดสรรคิวงานของ CPU ที่สม่ำเสมอต้องการการอ่าน/เขียนข้อมูลบน RAM ที่รวดเร็ว หากใช้ค่าสตรีมข้อมูลที่มีปัญหา Latency สูง การสลับงานของ Windows Quantum ก็ยังมีจุดหน่วงอยู่ดี แนะนำให้ใช้แรมคุณภาพสูงที่รองรับโปรไฟล์ XMP/EXPO เพื่อความเสถียรสูงสุด
   * 👉 [เลือกซื้อ RAM High-Speed DDR4/DDR5 ประสิทธิภาพสูงได้ที่นี่](https://shopee.co.th/search?keyword=DDR5+RAM+Gaming)

2. **ระบบระบายความร้อน CPU คุณภาพสูง (Premium CPU Cooler):**
   เมื่อ Windows ล็อคคิวงานแบบ Fixed Quantum ทำให้ CPU ทำงานได้เต็มประสิทธิภาพอย่างต่อเนื่องในรอบวิกฤต ซึ่งอาจส่งผลให้ความร้อนเพิ่มขึ้นชั่วขณะ หากระบบระบายความร้อนไม่ดีพอ CPU จะลดคล็อคสปีดตัวเอง (Thermal Throttling) ส่งผลให้เกิดอาการสะดุดทันที การใช้ชุดระบายความร้อนระดับพรีเมียมจึงเป็นสิ่งจำเป็น
   * 👉 [ดูรายละเอียดระบบระบายความร้อน CPU คุณภาพสูงได้ที่นี่](https://shopee.co.th/search?keyword=CPU+Cooler+Dual+Tower)

3. **สายสัญญาณเครือข่ายคุณภาพสูงลดปิง (Network Connection Quality):**
   บ่อยครั้งที่ผู้เล่นสับสนระหว่าง "Micro-stuttering จากเครื่องคอมพิวเตอร์" กับ "Network Jitter (อาการแลคจากสัญญาณอินเทอร์เน็ตสะดุด)" การเปลี่ยนมาใช้สายแลน Cat8 ที่มีเกราะป้องกันสัญญาณรบกวนจะช่วยเคลียร์ปัญหาการกระตุกที่เกิดจากคอขวดของสตรีมเน็ตเวิร์กได้อย่างเด็ดขาด
