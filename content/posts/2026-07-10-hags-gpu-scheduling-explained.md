# ถอดรหัส Hardware-Accelerated GPU Scheduling (HAGS): วิเคราะห์การจัดสรรคิว VRAM เพื่อลดภาระ CPU และเพิ่มประสิทธิภาพเฟรมเรต

ในโลกของการพัฒนาเกมและการรีดประสิทธิภาพระบบขีดสุด (System Optimization) ปัญหาคอขวด (Bottleneck) ที่เกิดขึ้นระหว่าง CPU และ GPU ถือเป็นโจทย์คลาสสิกที่วิศวกรพยายามแก้ไขมาโดยตลอด หนึ่งในฟีเจอร์ระดับสถาปัตยกรรมระบบปฏิบัติการที่ถูกออกแบบมาเพื่อทลายขีดจำกัดนี้คือ **Hardware-Accelerated GPU Scheduling (HAGS)** ซึ่งเปิดตัวครั้งแรกบน Windows 10 (เวอร์ชัน 2004) ร่วมกับ WDDM 2.7 (Windows Display Driver Model)

บทความนี้จะพาทุกคนไปเจาะลึกสถาปัตยกรรมการทำงานของ HAGS วิเคราะห์เชิงลึกเกี่ยวกับการจัดสรรคิวหน่วยความจำ (VRAM Scheduling) ผลกระทบต่อ Frame Latency และแนวทางการเปิดใช้งานอย่างปลอดภัยเพื่อให้ได้ประสิทธิภาพสูงสุด

---

## 1. เปรียบเทียบสถาปัตยกรรม: Traditional Scheduling vs. HAGS

เพื่อความเข้าใจที่ชัดเจน เราต้องย้อนกลับไปดูวิธีที่ระบบปฏิบัติการ Windows จัดการงานที่จะส่งไปประมวลผลบน GPU แบบดั้งเดิม เปรียบเทียบกับเมื่อเปิดใช้งาน HAGS

```
[แบบดั้งเดิม (Software Scheduling)]
CPU (OS Scheduler & Driver) ---> จัดคิวส่งคำสั่ง ---> [ VRAM / GPU ]
(เกิด Overhead สูงที่ CPU Thread, ละติจูดการส่งงานช้าลง)

[แบบใหม่ (HAGS - Hardware Scheduling)]
CPU ---> มอบหมายงานก้อนใหญ่ ---> [ Dedicated GPU Scheduler (Hardware) ] ---> ประมวลผลในตัว GPU เอง
(CPU ได้สิทธิ์ Thread คืน, ลด Latency ในการสลับบริบทการทำงาน)
```

### Traditional Scheduling (การจัดคิวแบบซอฟต์แวร์)
ในสถาปัตยกรรมแบบเก่า ระบบปฏิบัติการ Windows (ผ่าน High-Priority Thread ของ CPU) จะทำหน้าที่เป็น "ผู้จัดตารางงาน" (Scheduler) คอยรับคำสั่งกราฟิกจากแอปพลิเคชันหรือเกม นำมาจัดคิว จัดสรรทรัพยากร VRAM แล้วจึงทยอยส่งต่อให้ไดรเวอร์ GPU ทำงาน 

*   **ข้อจำกัด:** CPU ต้องคอยสลับบริบทการทำงาน (Context Switching) และรับภาระส่งข้อมูลเป็นระเบิดย่อยๆ (Submission Overhead) ส่งผลให้เกิด Latency สะสม และทำให้อัตราการสร้างเฟรมเรตขั้นต่ำ (1% Low FPS) ผันผวนเมื่อ CPU ตกอยู่ในสถานะ High Load

### Hardware-Accelerated GPU Scheduling (HAGS)
เมื่อเปิดใช้งาน HAGS หน้าที่การจัดลำดับคิวงาน (Scheduling) และการจัดการหน่วยความจำเสมือนของ GPU (VRAM Paging) จะถูกส่งผ่านไปยัง **ตัวประมวลผลร่วมพิเศษ (Dedicated Scheduler Co-processor)** ที่ฝังอยู่ในตัวชิป GPU โดยตรง (เช่น GSP หรือ GPU System Processor ของ NVIDIA)

*   **ผลลัพธ์:** CPU จะหลุดพ้นจากหน้าที่การเป็นตัวกลางคอยจัดคิว และหันไปทำหน้าที่ประมวลผล Logic ของเกม ฟิสิกส์ หรือ Network Thread แทน ขณะที่ GPU สามารถจัดการจัดเรียงคิวงานและดึงข้อมูลจาก VRAM ได้อย่างรวดเร็วในระดับฮาร์ดแวร์

---

## 2. เจาะลึกกลไกการจัดสรรคิว VRAM และ Command Buffer

เมื่อเปิดใช้งาน HAGS โครงสร้างการจัดการคิวของระบบจะเปลี่ยนไปในระดับ Memory Model ดังนี้:

### การจัดการ Paging ของ VRAM แบบไร้แรงเสียดทาน (Zero-Overhead Memory Paging)
ปกติแล้วเมื่อ VRAM ไม่เพียงพอ ระบบปฏิบัติการต้องทำหน้าที่ย้ายข้อมูล (Eviction) ระหว่าง VRAM และ System RAM (ผ่านบัส PCIe) ซึ่งกินพลังประมวลผลของ CPU สูงมาก แต่ภายใต้ HAGS หน่วยประมวลผลการจัดคิวฮาร์ดแวร์บน GPU จะควบคุม Memory Allocation Ring Buffer โดยตรง ทำให้การย้าย Page หน่วยความจำทำได้รวดเร็วกว่า และช่วยลดอาการสะดุด (Micro-stuttering) ได้อย่างเห็นได้ชัด

### ตารางเปรียบเทียบเชิงสถาปัตยกรรม

| ตัวบ่งชี้ทางเทคนิค | Traditional Software Scheduling | Hardware-Accelerated GPU Scheduling (HAGS) |
| :--- | :--- | :--- |
| **ตัวจัดการคิว (Scheduler)** | Windows OS Kernel (CPU Thread) | GPU Dedicated Coprocessor |
| **ความหน่วงในการตอบสนอง (Submit Latency)** | สูง (~2-5ms ขึ้นอยู่กับภาระของ CPU) | ต่ำมาก (<1ms ประมวลผลภายในวงจรปิดของ GPU) |
| **อัตราการใช้ CPU Overhead** | สูง (สะสมตามความซับซ้อนของเฟรมเรต) | ต่ำมาก (ลดการใช้งาน Core/Thread ลงอย่างชัดเจน) |
| **ความเสถียรของเฟรมเรต (Frame Pacing)** | มีโอกาสเกิด Micro-stutter หาก CPU ทำงาน 100% | เฟรมเรตนิ่งขึ้นอย่างสม่ำเสมอ (1% Low FPS ดีขึ้น) |

## 3. ผลกระทบต่อประสิทธิภาพในโลกจริง: ควรเปิดหรือปิด?

แม้ว่า HAGS จะเป็นเทคโนโลยีที่ยอดเยี่ยม แต่ประสิทธิภาพที่แท้จริงขึ้นอยู่กับสภาพแวดล้อมระบบของคุณ:

### ระบบที่ได้รับประโยชน์สูงสุด (Highly Recommended)
1.  **CPU-Bound Scenarios:** หากคุณเล่นเกมที่เน้นการประมวลผลของ CPU สูง (เช่น *Cyberpunk 2077*, *Microsoft Flight Simulator*, *Valorant*) หรือใช้งานการ์ดจอประสิทธิภาพสูงร่วมกับ CPU รุ่นกลาง 
2.  **การ์ดจอระดับเริ่มต้นและระดับกลาง:** การ์ดจอที่มี VRAM ขนาด 4GB - 8GB จะได้รับประโยชน์อย่างมากจากการจัดการ Memory Paging ที่มีประสิทธิภาพของ HAGS
3.  **การเปิดใช้งานฟีเจอร์ Frame Generation:** สำหรับคอเกมที่ต้องการใช้ **DLSS 3 (Frame Generation)** ของ NVIDIA หรือ **FSR 3** ของ AMD *จำเป็น* ต้องเปิดใช้ HAGS เนื่องจากอัลกอริทึมการแทรกเฟรมต้องการ Latency ในระดับฮาร์ดแวร์ที่ต่ำที่สุดเพื่อป้องกันภาพฉีกขาด

### ระบบที่อาจไม่เห็นผลต่าง หรือควรทดสอบก่อนใช้
*   **ระบบที่ GPU-Bound 100%:** หากเล่นเกมที่ความละเอียด 4K โดยที่ GPU ทำงานเต็มพิกัดแต่ CPU ทำงานต่ำมาก อาจจะไม่เห็นความแตกต่างของ FPS เฉลี่ย (Average FPS) แต่อาจช่วยเรื่องเฟรมไทม์ (Frame Time Consistency) ให้เรียบขึ้น
*   **โปรแกรมสตรีมมิ่งรุ่นเก่า:** ในอดีต ซอฟต์แวร์อย่าง OBS Studio (เวอร์ชันเก่ากว่า 27) อาจมีปัญหาการแย่งชิงทรัพยากรคิว GPU เมื่อเปิด HAGS ส่งผลให้เฟรมเรตในสตรีมตก แต่ในเวอร์ชันปัจจุบันได้รับการแก้ไข (Patched) เรียบร้อยแล้ว

## 4. คู่มือการเปิดใช้งาน HAGS อย่างปลอดภัยและแม่นยำ

ก่อนเริ่มดำเนินการ เพื่อความปลอดภัยสูงสุดต่อระบบปฏิบัติการและข้อมูลของคุณ เราขอแนะนำขั้นตอนการเตรียมตัวดังนี้

> [!WARNING]
> **ข้อควรระวังเพื่อความปลอดภัยสูงสุด (CRITICAL SAFETY):**
> การปรับแต่งระบบปฏิบัติการระดับลึกมีความเสี่ยงทำให้เกิดความไม่เข้ากันของไดรเวอร์ในบางระบบ ก่อนลงมือทำ กรุณาสร้าง **System Restore Point** เผื่อกรณีต้องการย้อนกลับระบบเดิม โดยทำตามขั้นตอนดังนี้:
> 1. กดปุ่ม `Windows` พิมพ์คำว่า **"Create a restore point"** แล้วกด Enter
> 2. ในหน้าต่าง System Properties คลิกเลือกไดรฟ์ระบบ (ปกติคือ C:) แล้วคลิกปุ่ม **"Create..."**
> 3. ตั้งชื่อจุดกู้คืนระบบ เช่น `Before_HAGS_Activation` แล้วกด **"Create"** เพื่อเซฟสถานะระบบไว้ก่อนเริ่มขั้นตอนถัดไป

### วิธีที่ 1: เปิดใช้งานผ่าน Windows Graphics Settings (แนะนำสำหรับผู้ใช้ทั่วไป)

1. คลิกขวาที่เดสก์ท็อป เลือก **Display settings**
2. เลื่อนลงมาด้านล่างสุด คลิกที่ **Graphics** (หรือ **Graphics settings** บน Windows 10)
3. ภายใต้หัวข้อ *Default settings* คลิกที่ลิงก์ **Change default graphics settings**
4. ทำการสับสวิตช์เปิดใช้งาน **Hardware-accelerated GPU scheduling** ให้เป็น **On**
5. **รีสตาร์ทเครื่องคอมพิวเตอร์** เพื่อให้ระบบเปิดทำงานโครงสร้าง WDDM ตัวใหม่

---

### วิธีที่ 2: เปิดใช้งานผ่าน Windows Registry (สำหรับแอดมินระบบและ Power Users)

ในกรณีที่คุณต้องการ Deploy ฟีเจอร์นี้ผ่านระบบสคริปต์ หรือต้องการเปิดใน Windows Edition ที่ไม่มี GUI เมนูดังกล่าว สามารถทำได้ผ่าน Registry Editor อย่างปลอดภัยตามขั้นตอนนี้:

```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers]
"HwSchMode"=dword:00000002
```

**คำอธิบายค่าคีย์ (Value Data):**
*   `HwSchMode = 1` -> ปิดการใช้งาน HAGS (Disabled)
*   `HwSchMode = 2` -> เปิดการใช้งาน HAGS (Enabled)

#### ขั้นตอนการตรวจสอบสถานะผ่าน PowerShell
คุณสามารถใช้คำสั่ง PowerShell ด้านล่างนี้เพื่อตรวจสอบสถานะปัจจุบันของระบบได้อย่างปลอดภัย โดยไม่ต้องเปิด Registry Editor ด้วยตนเอง:

```powershell
# ตรวจสอบว่าคีย์ HwSchMode ถูกตั้งค่าไว้เป็นระดับใด
$registryPath = "HKLM:\SYSTEM\CurrentControlSet\Control\GraphicsDrivers"
if (Test-Path $registryPath) {
    $value = Get-ItemProperty -Path $registryPath -Name "HwSchMode" -ErrorAction SilentlyContinue
    if ($value) {
        if ($value.HwSchMode -eq 2) {
            Write-Host "HAGS is currently: ENABLED (Value: 2)" -ForegroundColor Green
        } elseif ($value.HwSchMode -eq 1) {
            Write-Host "HAGS is currently: DISABLED (Value: 1)" -ForegroundColor Yellow
        } else {
            Write-Host "HAGS is in Windows Default State (Value: $($value.HwSchMode))" -ForegroundColor Cyan
        }
    } else {
        Write-Host "HAGS key does not exist. (System Default - Disabled)" -ForegroundColor Red
    }
}
```

## 5. อุปกรณ์เสริมฮาร์ดแวร์เพื่อปลดล็อกศักยภาพสูงสุดของระบบ

การมีซอฟต์แวร์ที่จัดสรรคิวได้รวดเร็วระดับนาโนวินาทีจะไม่มีประโยชน์เลย หากฮาร์ดแวร์ของคุณเจอกับปัญหาคอขวดในจุดอื่น เช่น ความร้อนสะสม หรือการถ่ายโอนข้อมูลจากฮาร์ดดิสก์แบบเก่า เพื่อการใช้งาน HAGS ควบคู่ไปกับการรีดประสิทธิภาพสูงสุด แนะนำให้อัปเกรดและใช้อุปกรณ์เสริมเหล่านี้:

1.  **SSD PCIe NVMe คุณภาพสูง:** เนื่องจากการทำ VRAM Paging ต้องการการสตรีม Texture ข้อมูลขนาดใหญ่จากหน่วยเก็บข้อมูลไปยัง VRAM อย่างรวดเร็ว การเปลี่ยนไปใช้ SSD ระดับท็อปจะช่วยลดเวลาการโหลดฉากและลดการแล็กได้อย่างมหาศาล  
    *   [ดูรายละเอียด SSD ความเร็วสูงและสั่งซื้อได้ที่นี่](https://shopee.co.th/search?keyword=NVMe+SSD+PCIe+4.0)
2.  **สายสัญญาณเชื่อมต่อประสิทธิภาพสูง (DisplayPort 1.4 / HDMI 2.1):** การส่งสัญญาณภาพที่มีรีเฟรชเรตสูงควบคู่ไปกับความละเอียดระดับ 2K/4K ต้องการสายนำสัญญาณที่มีคุณภาพเพื่อลดสัญญาณรบกวน ป้องกันปัญหาจอวูบดับขณะรันกราฟิกหนักๆ  
    *   [ดูรายละเอียดสาย HDMI 2.1 และ DisplayPort คุณภาพสูงได้ที่นี่](https://shopee.co.th/search?keyword=HDMI+2.1+DisplayPort+1.4+Cable)
3.  **ระบบระบายความร้อนตัวเครื่องและซิลิโคนคุณภาพสูง:** เมื่อ GPU ของคุณสามารถทำงานได้ต่อเนื่องและถูกคัดจัดตารางงานที่รวดเร็วขึ้น อุณหภูมิสะสมของชิปกราฟิกจะเพิ่มขึ้นอย่างต่อเนื่อง การใช้พัดลมเคสหรือชุดพัดลมระบายความร้อนสำหรับแล็ปท็อปที่ดีจึงจำเป็นมากในการป้องกัน Thermal Throttling  
    *   [ดูรายละเอียดอุปกรณ์ช่วยระบายความร้อนคอมพิวเตอร์และสั่งซื้อได้ที่นี่](https://shopee.co.th/search?keyword=Cooling+Pad+PC+Fan)

## บทสรุปเชิงสถาปัตยกรรม

**Hardware-Accelerated GPU Scheduling (HAGS)** ไม่ใช่เพียงแค่ฟีเจอร์เพิ่ม FPS แบบมหัศจรรย์ แต่เป็นการปรับโครงสร้างการส่งและจัดการคำสั่งกราฟิกระดับรากฐาน (Infrastructure Level Change) การเปลี่ยนผ่านหน้าที่นี้ช่วยลดการทำงานที่ซ้ำซ้อนของ CPU คืนทรัพยากรประมวลผลกลับมาให้ระบบ และลดละติจูดการสั่งงานภาพให้สั้นลงอย่างแท้จริง 

หากคุณเป็นเกมเมอร์ที่ต้องการเสถียรภาพเฟรมเรตขั้นสุด หรือต้องการเปิดใช้สถาปัตยกรรมการแทรกเฟรม (Frame Generation) ยุคใหม่ การเปิดใช้งาน HAGS ร่วมกับไดรเวอร์การ์ดจอเวอร์ชันล่าสุด คือขั้นตอนสำคัญที่คุณไม่ควรมองข้ามอย่างเด็ดขาด!
