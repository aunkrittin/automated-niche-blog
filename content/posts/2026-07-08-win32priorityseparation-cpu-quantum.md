# ไขความลับ Win32PrioritySeparation: ปรับแต่ง CPU Quantum เพื่อจัดสรรเวลาประมวลผลให้เกมและโปรแกรมสตรีมอย่างมีประสิทธิภาพสูงสุด

ในการเล่นเกมระดับแข่งขัน (eSports) หรือการสตรีมเกมควบคู่ไปด้วย (Single-PC Streaming) ปัญหาที่เกมเมอร์มักพบเจอไม่ใช่แค่เรื่องของเฟรมเรตเฉลี่ย (Average FPS) ที่ต่ำ แต่คือเรื่องของ **Frame Time instability** หรืออาการเฟรมไทม์แกว่ง, ไมโครสเตตเตอร์ (Micro-stuttering) และอาการภาพสะดุดเป็นช่วงๆ แม้ว่าสเปกเครื่องคอมพิวเตอร์จะสูงมากก็ตาม

สาเหตุหลักมักเกิดจากตัวจัดตารางงานของระบบปฏิบัติการ (Windows Thread Scheduler) จัดสรรเวลาการประมวลผลของ CPU (CPU Quantum) ให้กับกระบวนการทำงานต่างๆ (Processes) ได้ไม่สอดคล้องกับพฤติกรรมการทำงานในขณะนั้น บทความเชิงลึกนี้จะพาคุณไปเจาะลึกกลไกภายในของ Windows ที่เรียกว่า **Win32PrioritySeparation** เพื่อปรับแต่ง CPU Quantum ให้เหมาะสมกับระบบของคุณอย่างปลอดภัยและเห็นผลจริง

---

## 1. CPU Quantum คืออะไร? และ Windows จัดการงานอย่างไร?

สถาปัตยกรรมของ Windows เป็นระบบปฏิบัติการแบบ **Preemptive Multitasking** ซึ่งหมายความว่า CPU จะไม่ได้รันโปรแกรมใดโปรแกรมหนึ่งอย่างต่อเนื่องจนเสร็จ แต่จะสลับการทำงานระหว่างเธรด (Threads) หลายพันเธรดในระบบอย่างรวดเร็วเพื่อให้ดูเหมือนว่าทุกโปรแกรมทำงานพร้อมกัน

ช่วงเวลาที่แต่ละเธรดได้รับอนุญาตให้ประมวลผลบน CPU ก่อนที่จะโดนขัดจังหวะ (Interrupt) เรียกว่า **"Quantum"** 

ใน Windows Client (เช่น Windows 10 และ 11) ค่า Quantum จะถูกกำหนดให้มีขนาดสั้นและปรับเปลี่ยนได้ (Short and Variable) เพื่อให้แอปพลิเคชันที่อยู่บนหน้าจอหลัก (Foreground Window) ตอบสนองได้อย่างรวดเร็ว ในขณะที่ Windows Server จะใช้ค่า Quantum ขนาดยาวและคงที่ (Long and Fixed) เพื่อเน้นประสิทธิภาพในการประมวลผลงานเบื้องหลัง (Background Throughput)

## 2. เจาะลึกโครงสร้างบิตของ Win32PrioritySeparation

การควบคุมลักษณะของ CPU Quantum ใน Windows ถูกกำหนดผ่านคีย์เรจิสทรีที่ชื่อว่า `Win32PrioritySeparation` ซึ่งเป็นค่าเลขฐานสิบหก (Hexadecimal) ขนาด 32 บิต แต่ระบบจะสนใจเฉพาะ **6 บิตล่าง (Bits 0-5)** เท่านั้น โดยแบ่งโครงสร้างการทำงานออกเป็น 3 ส่วนดังนี้:

```
+-------------------+-------------------+-------------------+
|   Bits 4-5        |   Bits 2-3        |   Bits 0-1        |
|  (Priority Boost) |  (Quantum Length) |  (Quantum Type)   |
+-------------------+-------------------+-------------------+
```

### 1. Quantum Type (บิต 0-1): กำหนดความยาวของ Quantum
*   `01` (Short): เหมาะสำหรับเครื่อง Client เพื่อความลื่นไหลในการตอบสนองของ UI
*   `10` (Long): เหมาะสำหรับเครื่อง Server เพื่อรีดประสิทธิภาพสูงสุดจากหน่วยประมวลผลโดยลด Context Switching

### 2. Quantum Length (บิต 2-3): กำหนดรูปแบบความยืดหยุ่น
*   `01` (Variable): ความยาว Quantum ของแอปพลิเคชัน Foreground จะแปรผันตามสถานะ (ได้รับการ Boost พลังเพิ่ม)
*   `10` (Fixed): ความยาว Quantum คงที่เท่ากันหมด ไม่ว่าจะเป็นแอปพลิเคชันเบื้องหน้าหรือเบื้องหลัง

### 3. Priority Boost Ratio (บิต 4-5): อัตราส่วนการเพิ่มความสำคัญให้กับแอปพลิเคชันเบื้องหน้า
*   `00` (No Boost): ไม่มีการให้สิทธิ์พิเศษแก่หน้าต่าง Foreground
*   `01` (2:1 Ratio): หน้าต่าง Foreground ได้รับเวลาประมวลผลเป็น 2 เท่าของแอปทั่วไป
*   `10` (3:1 Ratio): หน้าต่าง Foreground ได้รับเวลาประมวลผลเป็น 3 เท่าของแอปทั่วไป

## 3. ผลกระทบต่อ "การเล่นเกม" และ "การสตรีมเกม"

การตั้งค่าเริ่มต้นของ Windows มักจะเปิดใช้งาน Foreground Boost เอาไว้สูงมาก ซึ่งส่งผลดีหากคุณใช้งานคอมพิวเตอร์เพื่อเปิดหน้าต่างเดียวเป็นหลัก แต่หากคุณทำสองสิ่งนี้พร้อมกัน:

*   **สถานการณ์ที่ 1: เล่นเกมสตรีมมิ่งบนคอมพิวเตอร์เครื่องเดียว (Single-PC Setup)**
    หากตั้งค่าเป็น Variable Quantum (มีการ Boost ไปที่เกมสูงมาก) CPU จะทุ่มเททรัพยากรเกือบทั้งหมดให้ตัวเกมที่อยู่หน้าจอหลัก จนทำให้โปรแกรมเบื้องหลังอย่าง OBS Studio, Discord หรือโปรแกรมเข้ารหัสวิดีโอ (Encoder) ขาดแคลน CPU Quantum ส่งผลให้เกิดอาการ **"สตรีมกระตุก/เฟรมตกในสตรีม"** ทั้งๆ ที่เฟรมเรตในเกมลื่นไหลปกติ
*   **สถานการณ์ที่ 2: เล่นเกมแนว eSports (Competitive Gaming)**
    ต้องการความหน่วง (Latency) ต่ำสุดขีด การใช้ Short Quantum จะช่วยให้ระบบสลับงานได้เร็ว แต่หากสลับบ่อยเกินไปก็จะเกิด Context Switch Overhead เพิ่มขึ้น ทำให้เฟรมไทม์ไม่นิ่ง

⚠️ **[WARNING ALERT] ข้อควรระวังและแนวทางความปลอดภัยสูงสุด**
การแก้ไขค่า Registry ในระบบปฏิบัติการ Windows มีความเสี่ยงหากกรอกค่าที่ผิดพลาดหรือไม่ได้มาตรฐานของเคอร์เนล (Kernel) อาจทำให้ระบบมีอาการค้าง เสถียรภาพลดลง หรือเกิดจอฟ้า (BSOD) ได้
1. **กรุณาสร้าง System Restore Point** ก่อนดำเนินการปรับแต่งทุกครั้ง
2. ห้ามป้อนค่าอื่นใดนอกเหนือจากค่ามาตรฐานที่แนะนำในบทความนี้เด็ดขาด
3. หากระบบมีอาการผิดปกติหลังการปรับแต่ง ให้กู้คืนค่ากลับเป็นค่าเริ่มต้น (Default) ทันที

## 4. ขั้นตอนการเข้าถึงและปรับแต่ง Win32PrioritySeparation

### วิธีที่ 1: การสำรองข้อมูลระบบ (สร้าง System Restore Point)
1. กดปุ่ม `Windows + S` พิมพ์คำว่า `Create a restore point` แล้วเลือกเปิดโปรแกรมขึ้นมา
2. คลิกที่ไดรฟ์ระบบ (ปกติคือ `C:`) แล้วกดปุ่ม **Create...**
3. ตั้งชื่อจุดกู้คืน เช่น `Before-Quantum-Tuning` แล้วกด **Create**

### วิธีที่ 2: การเปิดโปรแกรมแก้ไขเรจิสทรี (Registry Editor)
1. กดปุ่ม `Windows + R` พิมพ์ `regedit` แล้วกด Enter
2. คัดลอก Path ด้านล่างนี้ไปวางในช่อง Address Bar ด้านบนแล้วกด Enter:
   `Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl`
3. ในโฟลเดอร์นี้ คุณจะพบกับ Value ชื่อ `Win32PrioritySeparation`

## 5. ค่าแนะนำ (Presets) ที่เหมาะสมที่สุดสำหรับแต่ละการใช้งาน

เลือกค่าที่เหมาะกับพฤติกรรมการใช้งานของคุณ จากรายการที่ผ่านการคำนวณและทดสอบทางสถิติมาแล้วด้านล่างนี้:

| ลักษณะการใช้งาน (Scenario) | ค่าที่แนะนำ (Hexadecimal) | ค่าที่แนะนำ (Decimal) | อธิบายทางเทคนิค |
| :--- | :---: | :---: | :--- |
| **1. ค่าเริ่มต้นของ Windows (Default)** | `0x14` | `20` | Short, Variable, 3:1 Boost (เน้นการสลับหน้าต่างทำงานทั่วไป) |
| **2. เล่นเกมอย่างเดียวเน้นเฟรมไทมนิ่ง (eSports Pure Gaming)** | `0x26` | `38` | Long, Variable, 2:1 Boost (ลดแรงต้านใน CPU คอร์เดี่ยว ช่วยลด Micro-stuttering) |
| **3. เล่นเกมไปด้วย + สตรีมเกม/บันทึกหน้าจอไปด้วย (Gaming + Streaming)** | `0x18` | `24` | Short, Fixed, No Boost (เกมและ OBS ได้โควตาเวลา CPU เท่ากัน สตรีมไม่กระตุก เกมไม่หน่วง) |
| **4. พีซีสเปกสูง 12 Core+ เน้นประมวลผลหนัก (High-End Workstation)** | `0x28` | `40` | Long, Fixed, No Boost (โหมดเทียบเท่า Server รีดแบนด์วิดท์ CPU สูงสุด ลดการขัดจังหวะของระบบ) |

*หมายเหตุ: ค่าที่เหมาะสมที่สุดสำหรับผู้ใช้งานแบบ Hybrid (สตรีมด้วย เล่นเกมด้วย) คือ `0x18` หรือ `24` (Decimal)*

## 6. สคริปต์ลงทะเบียน (.reg) สำหรับนำไปใช้ทันที

เพื่อความปลอดภัยสูงสุดและป้องกันการป้อนค่าผิดพลาด คุณสามารถคัดลอกโค้ดด้านล่างนี้ไปเซฟเป็นไฟล์นามสกุล `.reg` เพื่อใช้งานได้ทันที

### สคริปต์สำหรับกรณี: เน้นเล่นเกมและสตรีมมิ่งพร้อมกัน (Short, Fixed, No Boost)
```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl]
"Win32PrioritySeparation"=dword:00000018
```

### สคริปต์สำหรับกรณี: เน้นเล่นเกมแบบแข่งขันดุดันสูงสุด (Short, Variable, High Boost)
```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl]
"Win32PrioritySeparation"=dword:00000026
```

### สคริปต์สำหรับคืนค่าเริ่มต้นของระบบ (Windows Default)
```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\PriorityControl]
"Win32PrioritySeparation"=dword:00000014
```

**วิธีใช้งาน:** คัดลอกโค้ดที่คุณต้องการไปวางในโปรแกรม Notepad จากนั้นกด Save As ตั้งชื่อไฟล์ว่า `quantum_tweak.reg` (ตรวจสอบให้มั่นใจว่านามสกุลลงท้ายด้วย `.reg` ไม่ใช่ `.txt`) จากนั้นดับเบิลคลิกที่ไฟล์เพื่อรันระบบ แล้วทำการ **Restart คอมพิวเตอร์** เพื่อให้การตั้งค่ามีผล

## 7. ตรวจสอบการทำงานของระบบหลังการปรับแต่ง

หลังจากปรับแต่งค่าและรีสตาร์ตระบบแล้ว คุณควรใช้เครื่องมือตรวจสอบเพื่อวัดผลลัพธ์เชิงตัวเลขดังนี้:

1. **CapFrameX หรือ MSI Afterburner:** ใช้บันทึกค่า Frametime ขณะเล่นเกมและสตรีมมิ่ง สังเกตว่ากราฟเฟรมไทม์มีความราบเรียบขึ้นหรือไม่ และค่า 0.1% Low FPS เพิ่มสูงขึ้นหรือไม่
2. **OBS Log Analyzer:** หลังจากทำการสตรีม ให้ส่ง Log File เข้าไปที่เครื่องมือวิเคราะห์ของ OBS เพื่อตรวจเช็คว่าไม่มีอาการ "Frames missed due to rendering lag" หรือ "Skipped frames due to encoding lag" ปรากฏขึ้นอีก
