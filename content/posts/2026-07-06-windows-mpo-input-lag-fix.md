## ไขความลับ Multi-Plane Overlay (MPO): ฮีโร่ผู้ปิดทองหลังพระของการเรนเดอร์บน Windows

สำหรับเกมเมอร์ที่ชอบเล่นเกมในโหมด **Borderless Windowed** เพื่อให้สลับหน้าจอ (Alt-Tab) ไปเปิดดิสคอร์ดหรือเบราว์เซอร์ได้อย่างรวดเร็ว คุณอาจเคยเจอปัญหาเฟรมเรตตก กระตุก หรือรู้สึกว่าเมาส์หน่วง (Input Lag) มากกว่าการเล่นแบบ Fullscreen ปกติ

แต่รู้หรือไม่ว่าใน Windows 10 และ 11 มีเทคโนโลยีระดับลึกชิ้นหนึ่งที่ถูกออกแบบมาเพื่อแก้ปัญหานี้โดยเฉพาะ นั่นคือ **Multi-Plane Overlay (MPO)** 

บทความนี้เราจะมาเจาะลึกสถาปัตยกรรมระบบเรนเดอร์ของ Windows วิเคราะห์ว่า MPO ทำงานอย่างไร และเหตุใดมันจึงเป็นตัวแปรสำคัญที่จะทำให้โหมด Windowed ของคุณลื่นไหลไม่ต่างจาก Exclusive Fullscreen พร้อมวิธีแก้ไขหากระบบนี้สร้างปัญหาให้กับเครื่องของคุณ

### DWM (Desktop Window Manager) และข้อจำกัดในการเรนเดอร์แบบเดิม

เพื่อที่จะเข้าใจความเจ๋งของ MPO เราต้องย้อนกลับไปดูสถาปัตยกรรมการจัดการหน้าจอของ Windows หรือ **Desktop Window Manager (DWM)** เสียก่อน

ในอดีต (ตั้งแต่ Windows Vista/7 เป็นต้นมา) DWM ทำหน้าที่เป็น "ตัวจัดองค์ประกอบภาพ" (Compositor) เมื่อคุณรันแอปพลิเคชันหรือเกมในโหมด Windowed:
1. แอปพลิเคชันแต่ละตัวจะวาดภาพลงใน Buffer (หน่วยความจำ) ของตัวเอง
2. DWM จะคัดลอก (Copy) ข้อมูลภาพจาก Buffer ของทุกแอปพลิเคชันมารวมกันใน Back Buffer ของระบบ
3. DWM ทำการ Composite (ผสมภาพ) แล้วส่งให้ GPU ส่งออกไปที่หน้าจอ

```
[Game Window Buffer]  ──┐
[Discord Buffer]      ──┼─> [DWM (Composite/Copy)] ──> [GPU Framebuffer] ──> [Display]
[Browser Buffer]      ──┘
```

**ผลเสียของระบบนี้คืออะไร?**
* **Input Lag เพิ่มขึ้น:** การที่ต้องรอ DWM รวบรวมและผสมภาพ ทำให้เกิดดีเลย์เพิ่มขึ้นอย่างน้อย 1-2 เฟรม (Double/Triple Buffering บังคับ)
* **เสียทรัพยากร GPU:** GPU ต้องเสียรอบการประมวลผล (Bandwidth) ไปกับการก๊อปปี้พิกเซลภาพในระดับระบบปฏิบัติการ
* **เฟรมเรตไม่นิ่ง:** หากแอปพลิเคชันเบื้องหลังมีการขยับหรือโหลด อาจส่งผลดึงเฟรมเรตของเกมให้ตกลงไปด้วย

### MPO คืออะไร? ฮาร์ดแวร์อ้อมสายตา (Hardware Bypass) ที่ช่วยลด Latency

**Multi-Plane Overlay (MPO)** คือฟีเจอร์ที่ถูกบรรจุเข้ามาในไดรเวอร์การแสดงผลยุคใหม่ (WDDM 1.3 ขึ้นไป) หน้าที่ของมันคือการยอมให้ GPU สั่งการให้หน้าจอวาดภาพแยกจากกันเป็น "เลเยอร์" (Planes) ในระดับฮาร์ดแวร์โดยตรง โดยไม่ต้องส่งข้อมูลผ่านกระบวนการคัดลอกพิกเซลของ DWM

ลองจินตนาการว่าหน้าจอของคุณคือแผ่นใสหลายๆ แผ่นซ้อนกัน:
* **Plane 0 (Background/DWM):** แสดงผลหน้าจอเดสก์ท็อป ไอคอน และแอปพลิเคชันอื่นๆ
* **Plane 1 (Foreground/Overlay):** แสดงผลหน้าต่างเกมของคุณโดยเฉพาะ

```
[Game (Direct to Hardware Overlay Plane)] ──┐
                                          ├─> [GPU Hardware Display Engine] ──> [Display]
[DWM / Desktop (Standard Plane)]        ──┘
```

เมื่อใช้ MPO ตัวคอนโทรลเลอร์แสดงผลของ GPU (Display Engine) จะหยิบภาพจาก Buffer ของเกมและ DWM มาสวมทับกันในระดับฮาร์ดแวร์ทันทีในจังหวะสแกนเอาต์พุต (Scan-out)

**ผลลัพธ์ที่ได้:**
* ** latency ต่ำเท่า Exclusive Fullscreen:** เพราะเฟรมของเกมถูกยิงตรงไปยังหน้าจอโดยไม่ต้องผ่านขั้นตอนผสมภาพของ DWM
* **G-Sync / FreeSync ทำงานได้สมบูรณ์แบบในโหมด Windowed:** อัตรารีเฟรชของจอจะซิงก์ตรงกับเฟรมเรตของเลเยอร์เกมโดยตรง ไม่ถูกดีเลย์จากเดสก์ท็อป
* **ประหยัดพลังงานและรีดเฟรมเรตสูงสุด:** ลดภาระการ Copy ของ GPU ทำให้ระบบมี Bandwidth เหลือไปปั่นเฟรมเรตในเกม

### เมื่อฮีโร่กลายเป็นผู้ร้าย: ทำไมบางคนถึงต้อง "ปิด" MPO?

แม้ว่าตามทฤษฎี MPO จะเป็นเทคโนโลยีที่ยอดเยี่ยม แต่ในความเป็นจริง ความเข้ากันได้ระหว่าง ไดรเวอร์การ์ดจอ (NVIDIA/AMD) และ Windows Update มักเกิดความไม่เสถียรอยู่บ่อยครั้ง

อาการผิดปกติที่มักเกิดขึ้นจาก MPO บกพร่อง:
1. **หน้าจอขาววาบ หรือกระพริบดำ (Flickering):** มักเกิดเมื่อใช้เว็บเบราว์เซอร์ที่เปิด Hardware Acceleration (เช่น Chrome, Edge) ควบคู่กับการเล่นเกม
2. **เฟรมเรตตกฮวบชั่วขณะ (Microstuttering):** เกิดขึ้นเมื่อมีการเปลี่ยนผ่านหน้าต่าง เช่น ป๊อปอัปแจ้งเตือนของ Discord เด้งขึ้นมาทับเลเยอร์เกม
3. **ระบบสั่นหรือค้าง (TDR - Timeout Detection and Recovery):** ไดรเวอร์การ์ดจอล่มและกู้คืนตัวเองขณะสลับหน้าจอ

### วิธีการเปิด-ปิด MPO อย่างปลอดภัย (Step-by-Step)

หากคุณกำลังเผชิญหน้ากับอาการจอกระพริบ ดับ หรือสแตตเตอร์ขณะสลับหน้าจอ การทดลองปิด MPO เป็นหนึ่งในวิธีแก้ปัญหามาตรฐาน (Workaround) ที่แม้แต่ทาง NVIDIA ก็แนะนำ

⚠️ **คำเตือนด้านความปลอดภัยที่สำคัญมาก (CRITICAL SAFETY WARNING)**
> การแก้ไข Registry ของระบบมีความเสี่ยง หากทำผิดพลาดอาจทำให้ระบบแสดงผลหรือ Windows ทำงานผิดปกติได้ 
> ก่อนเริ่มขั้นตอนใดๆ แนะนำให้คุณสร้าง **System Restore Point** เสมอ เพื่อให้สามารถย้อนกลับค่าเดิมได้หากเกิดปัญหา
> 
> **วิธีสร้าง System Restore Point:** 
> กดปุ่ม Start พิมพ์คำว่า `Create a restore point` -> คลิกปุ่ม `Create...` -> ตั้งชื่อ แล้วกด `Create` เป็นอันเสร็จสิ้น

#### วิธีที่ 1: การใช้ไฟล์สคริปต์ Registry (.reg) สำเร็จรูป (แนะนำและปลอดภัยที่สุด)

นี่คือโค้ดสคริปต์ลงทะเบียนที่ผ่านการตรวจสอบความถูกต้องตามมาตรฐานของ Microsoft และ NVIDIA

##### สคริปต์สำหรับ "ปิด" การทำงานของ MPO (Disable MPO)
ให้สร้างไฟล์ข้อความใหม่ใน Notepad คัดลอกโค้ดด้านล่างนี้ไปวาง แล้วบันทึกไฟล์เป็นชื่อ `disable_mpo.reg` (ต้องตรวจสอบให้แน่ใจว่านามสกุลไฟล์ลงท้ายด้วย `.reg` ไม่ใช่ `.txt`) จากนั้นดับเบิลคลิกเพื่อรันไฟล์และกดยอมรับการแก้ไข

```registry
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\Dwm]
"OverlayTestMode"=dword:00000005
```

##### สคริปต์สำหรับ "เปิด" การทำงานของ MPO (Enable MPO - ค่าเริ่มต้นของระบบ)
หากหลังจากปิดไปแล้วต้องการเปิดใช้งานใหม่อีกครั้งเพื่อประสิทธิภาพที่ดีขึ้น ให้ทำตามขั้นตอนเดิมด้วยโค้ดด้านล่างนี้ โดยบันทึกไฟล์เป็นชื่อ `enable_mpo.reg`

```registry
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\Dwm]
"OverlayTestMode"=-
```

*หมายเหตุ: หลังจากรันไฟล์ .reg เรียบร้อยแล้ว ให้ทำการ Restart คอมพิวเตอร์ของคุณ 1 ครั้งเพื่อให้ค่าใหม่มีผลใช้งาน*

### การปรับแต่ง Windows เพิ่มเติมเพื่อประสิทธิภาพการเล่นเกมสูงสุด

นอกเหนือจากการตั้งค่า MPO แล้ว หากคุณต้องการให้เกมในโหมด Windowed ลื่นไหลที่สุด คุณควรตั้งค่าเหล่านี้ควบคู่ไปด้วย:

1. **เปิดใช้งาน Optimizations for windowed games:**
   * ไปที่ `Settings` > `System` > `Display` > `Graphics` > `Default graphics settings`
   * ติ๊กเปิด **Optimizations for windowed games** (ฟีเจอร์นี้จะช่วยบังคับใช้สถาปัตยกรรมเรนเดอร์แบบใหม่ที่คล้ายกับ MPO และลด Input lag ในเกม DX10/DX11 รุ่นเก่า)

2. **เปิด Hardware-Accelerated GPU Scheduling (HAGS):**
   * ในหน้าเดียวกัน ให้เปิดใช้ **Hardware-Accelerated GPU Scheduling** เพื่อส่งมอบหน้าที่การจัดลำดับการทำงานให้ฮาร์ดแวร์ GPU จัดการโดยตรง ช่วยลดภาระของ CPU
